## Business Insights

- Local Whisper deployment avoids per-call API costs — viable for high-volume transcription workflows
- 3.5GB VRAM footprint is modest; runs on most modern GPUs alongside other workloads
- The two-input limitation (you vs. everyone else) is a pragmatic workaround but caps utility for multi-speaker meetings
- Growing market of voice-processing alternatives (Grok, Whisper, cloud APIs) signals commoditization of transcription

## Recommendations

- **Speaker diarization**: Explore `whisperx` or `pyannote` to get per-speaker labels beyond the 2-input workaround
- **Pipeline automation**: Wire the custom app into a post-processing step (summarization, action-item extraction) using an LLM
- **Cost tracking**: Compare local Whisper TCO (VRAM, power, maintenance) vs. API pricing at actual usage volumes
- **Multi-platform capture**: Consider OBS virtual audio or PulseAudio sinks for more flexible routing beyond VoiceMeeter's 2-input limit

**Market Gap:** Major players (Otter.ai, Fireflies, Tactiq, Krisp) all focus on Zoom/Teams/Google Meet. Almost none natively support Discord or system audio capture.

**Opportunity:** No polished desktop app captures system audio + mic + Discord with local transcription and speaker labels. Privacy-first, no cloud dependency.

### 2. Multi-Speaker Separation from Discord

**Key Insight:** Discord bots can receive **per-user audio streams** via the Discord voice API. Each user's audio arrives as a separate Opus stream — **no diarization needed**, speakers are pre-separated at the protocol level.

**Existing Tools:**
- **Craig Bot** — Records Discord calls with per-user tracks but no real-time transcription
- No widely-known tool does real-time per-speaker transcription in Discord

**Opportunity:** A bot that joins voice, receives individual streams, transcribes each with Whisper Turbo/Parakeet locally, and outputs labeled real-time transcripts.

### 3. Local vs Cloud Transcription

| Service | Cost/minute | Diarization | Real-time |
|---------|-------------|-------------|-----------|
| OpenAI Whisper API | $0.006 | No | No |
| Deepgram Nova-3 | ~$0.004-0.015 | Yes | Yes |
| AssemblyAI | ~$0.01-0.03 | Yes | Yes |
| Local Whisper | $0 (after GPU) | Via pyannote | Yes |

**Break-even:** At $0.006/min API cost, 2 hours/day = ~$22/month. A used RTX 3060 (~$200) breaks even in ~9 months. Heavy use (8+ hrs/day) = 3 months.

### Overall Assessment

The **most promising build** combines all three: a desktop app capturing system audio + mic, Discord bot integration for per-speaker separation, and local transcription with Parakeet TDT or Whisper Turbo. Key differentiators:

1. **Discord-native** — no existing tool does real-time Discord transcription with speaker labels
2. **Privacy-first local processing** — no audio leaves the machine
3. **System audio capture** — works with any app, not just meeting platforms
4. **Per-speaker transcripts** from Discord (leveraging protocol's built-in separation)

Target market: streamers, podcast hosts, D&D groups, gaming communities. Minimal competition.
